# Bitácora de Aprendizaje - Git y GitHub
## Concepto del día: [Configuración e Inicialización de Repositorio Local y Remoto]

### 1. ¿Qué problema resuelve exactamente?
Permite mantener un historial de versiones de mi código en la computadora local, tener un respaldo seguro en GitHub y poder sincronizar mi trabajo entre mi PC actual y mi futura laptop usando una misma cuenta.

### 2. ¿Cuáles son sus límites o cuándo NO debo usarlo?
* Git no sube los cambios de forma automática; se debe ejecutar manualmente la secuencia `git add .` -> `git commit` -> `git push`.
* GitHub ya no acepta contraseñas de usuario normales en la terminal por seguridad; requiere configurar un Token de Acceso Personal (PAT).

### 3. Explicación simple (Técnica Feynman):
* **Git** es la cámara que toma fotos (`git commit`) de los archivos en mi PC.
* **GitHub** es la galería en la nube donde guardo esas fotos (`git push`).
* Puedo firmar las fotos como `"Victor Vasquez (PC)"` usando el mismo correo registrado en GitHub para saber desde qué equipo programé, sin perder la autoría en mi perfil.

### 4. ¿Cómo lo rompí y qué error dio?
* **Al ejecutar `git init`:** Arrojó una advertencia (`hint`) recomendando cambiar la rama por defecto de `master` a `main`.
  * *Solución:* Usé `git config --global init.defaultBranch main` y `git branch -m main`.
* **Al intentar salir de `git log`:** La terminal se quedó bloqueada en el historial.
  * *Solución:* Aprendí que se sale presionando la tecla `q` (Quit).
* **Al hacer `git push` por primera vez:** Puse mi contraseña de GitHub y me arrojó `remote: Invalid username or token. Password authentication is not supported`.
  * *Solución:* Creé un Personal Access Token (PAT) en GitHub con permisos de `repo` y usé dicho token como contraseña en la terminal.
