# Bitácora de Aprendizaje - Git y GitHub
## Concepto del día: [Control de Versiones y Sincronización]

### 1. ¿Qué problema resuelve exactamente?
Permite guardar el historial de cambios de mi código (puntos de restauración), tener un respaldo seguro en la nube (GitHub) y trabajar sincronizado entre mi PC y mi futura laptop sin perder archivos.

### 2. ¿Cuáles son sus límites o cuándo NO debo usarlo?
* Git no sube los cambios automáticamente a la nube; requiere que yo ejecute explícitamente `git add`, `git commit` y `git push`.
* No debo subir archivos temporales, basura del sistema o contraseñas a GitHub (para eso se usa el archivo `.gitignore`).

### 3. Explicación simple (Técnica Feynman):
* **Git** es como la cámara fotográfica en mi PC que toma fotos (`commits`) de mi proyecto.
* **GitHub** es como un álbum de fotos en la nube donde guardo copias de esas fotos.
* Puedo firmar las fotos como "Victor (PC)" o "Victor (Laptop)" usando el mismo correo de GitHub para saber desde qué equipo trabajé.

### 4. Duda resuelta / ¿Cómo lo entendí?
* **Duda:** ¿Debo usar la misma cuenta en la PC y Laptop? 
* **Solución:** Sí, el correo (`user.email`) debe ser el mismo registrado en GitHub para que reconozca mi autoría, pero el nombre (`user.name`) lo puedo personalizar en cada máquina para saber desde cuál programé.
