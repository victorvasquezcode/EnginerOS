# Bitácora de Aprendizaje - Lógica de Programación
## Concepto del día: [Estructuras de Control - Bucle For y Función range()]

### 1. ¿Qué problema resuelve exactamente?
Permite repetir un bloque de código un número determinado de veces o recorrer elemento por elemento una secuencia (como un texto o una lista) sin necesidad de duplicar líneas de código.

### 2. ¿Cuáles son sus límites o cuándo NO debo usarlo?
* No debo usar la sintaxis clásica de C/Java `(i=0; i<10; i++)` con comas o puntos y comas; Python usa la palabra reservada `in` junto con `range()` o colecciones.
* `range(fin)` NUNCA incluye el número final. Siempre se detiene un paso antes (`range(8)` llega hasta el `7`).
* No debo usar un `for` si no conozco de antemano cuántas veces se debe repetir el ciclo o si la parada depende de una condición dinámica externa (para eso se usa un bucle `while`).

### 3. Explicación simple (Técnica Feynman):
* El bucle `for` es como un inspector que va revisando elemento por elemento en una fila.
* La variable iteradora (`variable` o `i`) es la caja temporal que guarda el elemento actual de la vuelta.
* `range(inicio, fin, paso)` es un generador de secuencias numéricas. Sirve como el marcador de vueltas del bucle.
* Si le pasamos un string (`"Vasquez"`), el `for` lo recorre letra por letra.

### 4. ¿Cómo lo rompí y qué error dio?
* **Intenté usar sintaxis de C++ en el `for`:** Escribí `for (i > 8 , i++ , i = 0) in`. Esto causó un `SyntaxError` porque Python no usa paréntesis ni comas para configurar ciclos.
  * *Solución:* Reemplacé la estructura por `for variable in range(8):`.
* **Confusión con el valor impreso:** Al colocar `a = 10` dentro de `for variable in range(8):` e imprimir `a`, el programa imprimió `10` ocho veces porque no estaba imprimiendo la variable iteradora del ciclo.
  * *Solución:* Imprimir directamente `variable` para ver la progresión de la secuencia.
