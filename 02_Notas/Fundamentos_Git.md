# Glosario de Conceptos de Git y GitHub

* **Git:** Sistema de control de versiones local que rastrea los cambios en los archivos.
* **GitHub:** Servicio en la nube que aloja repositorios de Git remotos.
* **Repositorio (Repo):** Carpeta administrada por Git que contiene el historial de cambios del proyecto.
* **Commit:** Guardado o "fotografía" del estado del código en un momento específico con un mensaje descriptivo.
* **Staging Area:** Zona intermedia donde preparas los archivos (`git add`) antes de hacer un commit.
* **Branch (Rama):** Línea independiente de desarrollo (la principal suele llamarse `main`).

## Comandos Esenciales
* `git status`: Muestra qué archivos han sido modificados o agregados.
* `git add .`: Prepara todos los cambios para el guardado.
* `git commit -m "mensaje"`: Guarda los cambios localmente con una descripción.
* `git push`: Sube los guardados locales al repositorio de GitHub.
* `git pull`: Descarga los cambios de GitHub a la computadora (útil para cuando trabajes en tu laptop).
